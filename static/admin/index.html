Content administration

Edit the lab website

Pages CMS provides structured forms for the lab's research, people, publications, projects and news. Every saved change is recorded in GitHub.

Open Pages CMS Repository: cwanglab/cwanglab.github.io   Branch: main

First-time setup

  1. Sign in with GitHub

    Use an account with access to the cwanglab organisation.

  2. Authorise the repository

    Install Pages CMS for cwanglab/cwanglab.github.io only.

  3. Open main

    Select the main branch. The editor reads its forms from .pages.yml.

Editor areas

Site settingsLab identity, logo, mission, address and academic profile links.
Research directionsResearch summaries, full descriptions, figures, captions and source links.
PeopleMembers, positions, biographies, photographs and academic profiles.
PublicationsVerified bibliographic metadata and DOI, open-access, code and data links.
Projects and codeProject status, summaries, research areas, images and outputs.
NewsPublic announcements with a draft switch for unfinished records.
Social queueNon-confidential X, LinkedIn and website copy with review status.

Access

Site administratorA named GitHub account with repository access and two-factor authentication. Can manage content, configuration and collaborators.
Content editorInvited by email in Pages CMS. Can edit content and media without repository administration.

This page is an entry point, not a password form. GitHub Pages is static; secure editing and saving are handled by Pages CMS and GitHub authentication.

Content changes should be checked on the public site after deployment.
